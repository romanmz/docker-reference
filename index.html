Docker Reference

Dependencies

  • Docker
  • Docker Compose
  • Docker Machine
  • VirtualBox (for creating swarms with virtual machines)

Concepts

Container:
Base of the application, everything inside is virtualized and isolated from the host (except the ports and shared resources you define). The Dockerfile file defines what needs to go inside the container.
Service:
In a distributed application there’s often separate pieces that perform different actions (e.g. storing data on the database, converting uploaded videos, streaming data, etc…), each of these pieces is called a “service”. A single service can run one or more containers (all from the same image) by enabling load balancing. The service uses a docker-compose.yml file to define how many containers it needs and how they need to behave in production.
Task:
Each individual container running in a service is called a “task”, every time the service performs an action it could be run by a different task from the same service.
Swarm:
A swarm is a cluster of multiple machines running the same application through Docker. The machines in a swarm can be physical or virtual.
Swarm Manager:
Takes your commands and applies them to all the machines in a swarm. You can specify the logic the swarm manager will use to distribute the containers (equal number, focus on the least busy machines, etc).
Workers:
Extra machines that can join a swarm to help perform tasks (if allowed by the swarm manager).
Node:
Each individual machine running within a swarm.
Stack:
A stack is a group of interrelated services that share dependencies and can be scaled together. A simple application can have a single stack, more complex apps can use multiple stacks.
Portable image:
The specific OS/runtime, version number, and configuration that will run the application.
Repository:
Collection of images.
Registry:
Collection of repositories.
